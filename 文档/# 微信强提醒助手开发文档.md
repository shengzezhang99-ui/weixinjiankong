# 微信强提醒助手开发文档

## 1. 项目背景

个人微信 PC 端消息提醒不够明显，尤其在群聊中被 `@` 或出现紧急关键字时，容易错过重要消息。因此需要开发一个运行在 Windows 电脑端的辅助软件，用于监控微信消息提醒，并在命中特定规则时发出强提醒。

由于个人微信 PC 端的新消息提醒不一定进入 Windows 系统通知流，因此本项目不采用 Windows 通知中心监听方案，而采用：

> **监听微信自身弹窗窗口 + 截图 OCR 识别 + 规则匹配 + 本地强提醒 + 超时升级通知**

---

## 2. 项目目标

开发一个 Windows 桌面软件，后台常驻运行，自动监控微信 PC 端的新消息弹窗。当指定微信群出现 `@我` 或包含配置的关键字时，软件发出强烈提醒。如果用户在设定时间内未确认处理，则通知指定联系人，让对方电话提醒用户。

---

## 3. 核心使用场景

### 场景一：群内有人 @ 我

微信群弹窗内容：

```text
运维值班群
王五：@张三 线上报警了，看一下
```

软件识别到：

```text
群名 = 运维值班群
内容包含 @张三
```

触发：

```text
电脑播放强提醒声音
弹出置顶提醒窗口
开始倒计时
```

如果 3 分钟内用户未点击“已处理”，则发送通知给指定联系人。

---

### 场景二：群内出现关键字

微信群弹窗内容：

```text
项目A群
李四：生产环境支付失败，疑似故障
```

软件识别到：

```text
群名 = 项目A群
内容包含 支付失败 / 故障
```

触发强提醒。

---

### 场景三：用户未及时处理

触发强提醒后，软件显示：

```text
将在 3 分钟后通知李四
```

如果用户没有点击“已处理”，系统通过模拟点击通知联系人。

---

## 4. 第一版范围

### 4.1 第一版必须实现

1. 后台托盘常驻。
2. 检测微信 PC 端弹出的新消息提示窗口。
3. 对微信弹窗截图。
4. 使用 OCR 识别弹窗文字。
5. 支持配置：
   - 监控群名；
   - 我的 @ 名称；
   - 关键字；
   - 超时时间；
   - 联系人通知方式。
6. 命中规则后播放强提醒声音。
7. 弹出置顶提醒窗口。
8. 用户可点击：
   - 已处理；
   - 延迟 5 分钟；
   - 立即通知联系人。
9. 超时未处理时，通知指定联系人。
10. 本地保存触发日志。

---

### 4.2 第一版暂不实现

以下功能第一版不建议做：

1. 不读取微信本地数据库。
2. 不注入微信进程。
3. 不破解或逆向微信协议。
4. 不自动操作微信发送消息。
5. 不自动判断用户是否真的在群里回复。
6. 不保证监控所有未弹窗的群消息。

第一版以“易用、稳定、低风险”为主。

# 7. 模块设计

## 7.1 窗口监听模块

### 目标

检测微信 PC 端弹出的新消息提示窗口。

### 实现思路

通过 Win32 API 枚举或监听窗口。

需要识别：

```text
1. 窗口所属进程为 WeChat.exe
2. 窗口可见
3. 窗口尺寸较小
4. 窗口不是微信主窗口
5. 窗口位置通常在屏幕右下角或右侧
6. 窗口出现时间较短
```

### 推荐方式

第一版可以使用轮询方式：

```text
每 100ms - 300ms 枚举一次桌面窗口
发现新的微信小窗口后处理
```

后续优化为：

```text
SetWinEventHook 监听窗口显示事件
```

### 需要过滤的窗口

不要处理：

```text
微信主窗口
聊天窗口
登录窗口
文件传输窗口
图片查看窗口
设置窗口
已经处理过的弹窗
```

---

## 7.2 截图模块

### 目标

对微信弹窗区域进行截图，供 OCR 识别。

### 实现方式

可使用：

```text
mss
PIL.ImageGrab
win32gui + win32ui
```

建议：

1. 获取窗口坐标。
2. 对该窗口区域截图。
3. 将图像放大 1.5 到 2 倍。
4. 转灰度。
5. 增强对比度。
6. 送入 OCR。

---

## 7.3 OCR 识别模块

### 推荐 OCR

```text
PaddleOCR
```

原因：

- 中文识别效果较好；
- 支持本地离线；
- 不依赖云服务；
- 对隐私更友好。

### OCR 输入

微信弹窗截图。

### OCR 输出

原始文本列表，例如：

```json
[
  "运维值班群",
  "王五：@张三 线上报警了，看一下"
]
```

### 识别后处理

需要做文本清洗：

```text
1. 去除多余空格
2. 统一全角半角
3. 去除换行
4. 修正常见 OCR 误识别
5. 进行模糊匹配
```

---

## 7.4 消息解析模块

### 输入

OCR 识别文本。

### 输出

标准结构：

```json
{
  "chat_name": "运维值班群",
  "sender": "王五",
  "content": "@张三 线上报警了，看一下",
  "raw_text": "运维值班群\n王五：@张三 线上报警了，看一下",
  "time": "2026-07-13 10:30:20"
}
```

### 解析策略

微信弹窗格式可能有多种，解析不能过于死板。

常见格式：

```text
群名
发送人：消息内容
```

或者：

```text
群名
消息内容
```

或者 OCR 结果合并成：

```text
运维值班群 王五：@张三 线上报警了，看一下
```

需要支持容错。

## 7.5 规则匹配模块

### 配置项

```json
{
  "monitored_chats": [
    "运维值班群",
    "项目A群"
  ],
  "my_names": [
    "@张三",
    "@三哥",
    "@zhangsan"
  ],
  "keywords": [
    "报警",
    "宕机",
    "P0",
    "紧急",
    "线上故障",
    "支付失败"
  ],
  "escalation_minutes": 3
}
```

### 触发条件

只要满足以下任一条件即可触发：

```text
1. 指定群内出现 @我
2. 指定群内出现关键字
```

### 伪代码

```python
def should_alert(message, config):
    chat_name = message["chat_name"]
    content = message["content"]
    raw_text = message["raw_text"]

    if not is_monitored_chat(chat_name, raw_text, config.monitored_chats):
        return False

    if any(name in raw_text for name in config.my_names):
        return True

    if any(keyword in raw_text for keyword in config.keywords):
        return True

    return False
```

## 7.6 模糊匹配设计

OCR 可能识别错误，例如：

```text
运维值班群 → 运维值班君
报警 → 报警！
P0 → PO
```

建议加入模糊匹配。

### 可选算法

```text
difflib.SequenceMatcher
rapidfuzz
Levenshtein distance
```

### 建议规则

```text
群名相似度 >= 85% 视为匹配
关键字优先精确匹配，必要时再模糊匹配
@名称优先精确匹配
```

## 7.7 强提醒模块

### 触发后的动作

1. 循环播放报警音。
2. 弹出置顶窗口。
3. 窗口闪烁。
4. 显示倒计时。
5. 托盘图标变色或闪烁。

### 报警窗口内容

```text
微信强提醒

群：运维值班群
内容：@张三 线上报警了，看一下

已响铃：00:25
将在：02:35 后通知李四

[我已处理] [延迟5分钟] [立即通知李四]
```

### 按钮行为

#### 我已处理

```text
停止报警声音
关闭提醒窗口
将本次提醒标记为已处理
写入日志
```

#### 延迟 5 分钟

```text
停止当前倒计时
增加 5 分钟
继续提醒或间隔提醒
```

#### 立即通知联系人

```text
立即调用通知接口
记录日志
可继续响铃，直到用户点击已处理
```

# 11. 关键判断逻辑

## 11.1 是否为微信窗口

```text
进程名 == WeChat.exe
```

---

## 11.2 是否为疑似弹窗

满足多个条件：

```text
1. 可见窗口
2. 窗口面积小于微信主窗口
3. 宽高在合理范围内
4. 位于屏幕右下区域或右侧区域
5. 非主窗口标题
6. 最近新出现
```

建议不要只靠单一条件。

---

## 11.3 是否重复处理

微信弹窗可能被多次枚举到，需要去重。

去重方式：

```text
1. hwnd 去重
2. OCR 原始文本 + 时间窗口去重
3. 同一群同一内容 30 秒内只触发一次
```

示例：

```text
如果 raw_text 相同，并且 30 秒内已处理过，则忽略。
```

---

## 11.4 是否命中群名

优先策略：

```text
OCR 文本中包含配置的群名
```

增强策略：

```text
使用模糊匹配，相似度 >= 85%
```

如果识别不到群名，可以配置一个选项：

```text
强关键字无群名也提醒：开启/关闭
```

建议默认关闭，避免误报。

# 13. 异常处理

## 13.1 微信未运行

提示：

```text
未检测到微信正在运行，请打开微信。
```

托盘状态显示黄色。

---

## 13.2 OCR 初始化失败

提示：

```text
OCR 初始化失败，请检查模型文件或运行环境。
```

---

## 13.3 通知联系人失败

处理：

```text
1. 记录错误日志
2. 重试 2 次
3. 仍失败则在本地强提醒窗口显示失败原因
```

---

## 13.4 微信弹窗识别失败

处理：

```text
1. 保存截图到 debug 目录，方便调试
2. 记录 OCR 原文
3. 不触发报警
```

可在设置中开启：

```text
调试模式：保存未识别弹窗截图
```

# 15. 权限与隐私

本软件会截取微信弹窗内容并进行本地 OCR。

要求：

```text
1. 默认本地处理，不上传微信截图。
2. 日志中可以保存 OCR 文本，但应允许用户关闭。
3. 调试截图默认不保存，只有开启调试模式才保存。
4. 配置文件中的 token 不应明文展示在 UI 中。
```

# 18. 风险与限制

## 18.1 主要限制

```text
1. 微信不弹窗时无法检测到消息
2. 群开启免打扰后可能无弹窗
3. 微信弹窗不显示详情时无法识别内容
4. OCR 可能误识别
5. 微信版本更新可能改变弹窗样式
6. 多显示器和高 DPI 环境需要额外适配
```

---

## 18.2 规避建议

```text
1. 重点群不要开启免打扰
2. 微信设置中开启消息详情显示
3. 使用大字体或默认缩放比例
4. 第一版加入调试截图功能
5. 允许用户手动测试 OCR
6. 关键字尽量配置多个近义词
```

---

# 19. 第一版验收标准

满足以下条件即可认为第一版可用：

```text
1. 软件可在 Windows 上启动并托盘常驻
2. 能检测到微信新消息弹窗
3. 能对弹窗截图并 OCR 识别中文内容
4. 能根据配置的群名、@名称、关键字触发提醒
5. 提醒声音能够循环播放
6. 提醒窗口置顶显示
7. 用户点击“已处理”后停止提醒
8. 超过设定时间未处理时能通知联系人
9. 配置能保存和重新加载
10. 日志能记录关键流程
```

---

# 20. 推荐第一版最终方案

第一版建议明确为：

```text
微信弹窗 OCR 监控版
```

核心原则：

```text
不破解微信
不读微信数据库
不注入微信
不自动发微信消息
只监控微信弹窗
本地 OCR
本地强提醒
超时通知联系人
```

这样实现难度适中，个人使用方便，风险和维护成本相对较低。
