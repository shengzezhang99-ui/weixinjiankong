# 微信强提醒助手工作交接摘要

更新时间：2026-07-21

## 1. 当前结论

当前主线已经稳定为：

```text
自动窗口 OCR + 固定区域 OCR 兜底 + 规则匹配 + 本地强提醒
```

UIA 监控已移除，不再作为正式功能入口。

## 2. 已完成内容

### 2.1 界面重构

- 左侧导航已收敛为：
  - 规则配置
  - 监控设置
  - 通知报警
  - 测试工具
  - 运行日志
- `监控设置` 内使用标签页切换：
  - 自动窗口 OCR
  - 固定区域 OCR
- `定位微信窗口` 的重复入口已去掉，保留窗口下拉框。
- 自动窗口 OCR 页上的按钮已做颜色区分，`框选消息区` 是高亮操作。

### 2.2 监控能力

- 自动窗口 OCR：
  - 支持按进程名和窗口标题定位微信窗口
  - 支持下拉选择具体监控窗口
  - 支持截取窗口预览
  - 支持在窗口内框选消息区
  - 支持测试 OCR
- 固定区域 OCR：
  - 保留为兜底模式
  - 支持框选区域和单次测试
- 去重方式：
  - 按行去重，不再只依赖整段文本 hash
  - 支持启动时忽略当前历史文本
  - 支持去重秒数配置

### 2.3 强提醒与通知

- 强提醒仍使用 Flet 内嵌弹窗。
- 已修复点击空白导致弹窗意外消失的问题，当前弹窗为 `modal=True`。
- 通知方式只保留模拟点击通知。
- 已补通知并发保护：
  - `notify_inflight` 防止同一个 alert 重复拉起通知线程
  - 通知成功后才置为已通知
  - 重复点击“立即通知”会被忽略
  - 通知失败后按配置自动重试，达到上限后停止自动重试

### 2.4 运行清理

- 新增截图自动清理逻辑，按保留张数控制 `screenshots_debug`。
- 新增日志/截图清理按钮。
- 日志使用轮转写入。

## 4. 已知问题和注意点

- 自动窗口 OCR 仍受窗口布局影响，边距需要按实际微信窗口微调。
- OCR 文本质量会直接影响规则匹配，尤其是 `@` 和群名。
- 模拟点击通知依赖窗口坐标，联系人窗口移动后可能失效。
- 模拟点击通知会真的执行鼠标点击和粘贴。

## 8. 本轮开发进展

完成时间：2026-07-21

- 新增 `wechat_alert_assistant/dedup.py`，按自动窗口、固定区域分别维护去重状态。
- OCR 行由精确 hash 判重升级为可配置相似度判重，默认阈值 `0.88`。
- 当前截图中持续存在的文本会刷新去重时间，不再因时间到期重复报警。
- 新增自动窗口 OCR 与固定区域 OCR 的跨来源提醒合并，默认窗口 `12` 秒。
- @ 名称支持模糊识别，已覆盖 `SI/Sl` 一类轻微 OCR 波动。
- 处理确认会清理活动 Alert；已经处理的 Alert 不再被迟到的异步通知结果改写状态。
- 自动窗口测试发现裁剪面积超过窗口 85% 时会显示风险提示。
- 新增 `tests/` 回归测试、脱敏配置示例和 `.gitignore`。
- 新增通知失败自动重试配置与状态保护，默认失败后再试 2 次、间隔 60 秒。

下一步现场验证重点：

1. 重启应用加载新代码，用真实微信消息验证自动窗口与固定区域同时启用时只提醒一次。
2. 重新框选自动窗口消息列表，排除标题栏、聊天输入框和发送按钮。
3. 更换当前无效的报警音文件。
4. 观察不同字体和 DPI 下 `0.88` 阈值的误报、漏报情况，再根据日志调整。
5. 用隔离的模拟点击窗口验证通知失败重试与成功停止行为。
